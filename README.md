# LifeOS Weekly Tech Log 2025  This repository records my weekly engineering practice. Each week includes: - code - engineering log (log.md) - summary (readme.md)
============================================================
程式是由什麼組成？（C / BMC 版本 v1.0）
============================================================

關鍵字:四大核心構成
內容:
  1) 資料型態 (Data Types)
  2) 變數 (Variables)
  3) 函數 (Functions)
  4) 控制流程 (Control Flow)

------------------------------------------------------------
1) 資料型態 Data Types
------------------------------------------------------------
關鍵字:定義
內容:
  - 描述「這個資料長什麼樣子、能存什麼」

關鍵字:基本型態
內容:
  - int        : 整數（fd、gpio 編號、迴圈計數）
  - char       : 單一字元（'0', '1', '\n'）
  - char[]     : 字串／路徑（"/sys/class/gpio/..."）
  - uint8_t    : 8-bit 無號整數（I2C/SMBus buffer）
  - struct foo : 自訂結構（LED 資訊、pattern）
  - enum foo   : 自訂列舉（LED 狀態、錯誤碼）

------------------------------------------------------------
2) 變數 Variables
------------------------------------------------------------
關鍵字:定義
內容:
  - 「套用某個資料型態的實際一塊記憶體」

關鍵字:範例
內容:
  - int fd;                 → 檔案描述子
  - int gpio = 37;          → 第 37 號 GPIO
  - char path[64];          → 存放路徑字串
  - struct LedInfo led0;    → 一顆 LED 的資訊

關鍵字:作用
內容:
  - 承接函數運算結果
  - 在控制流程中保存狀態（迴圈次數、目前模式）

------------------------------------------------------------
3) 函數 Functions
------------------------------------------------------------
關鍵字:定義
內容:
  - 一段可以被重複呼叫的「動作」

關鍵字:型態 + 參數 + 回傳
內容:
  - 型態: 函數回傳值（int / void / struct foo）
  - 參數: 呼叫時丟進來的資料
  - 回傳: 把結果丟回呼叫者

關鍵字:範例
內容:
  - int main(int argc, char *argv[])
  - int GpioSetValue(int gpio, int value)
  - int WriteOnce(const char *path, const char *value)

關鍵字:角色
內容:
  - 把複雜問題拆成小步驟
  - 讓程式結構清楚，可讀性高
  - 未來可以重複使用（library / daemon）

------------------------------------------------------------
4) 控制流程 Control Flow
------------------------------------------------------------
關鍵字:定義
內容:
  - 決定「程式接下來走哪一條路」

關鍵字:主要語法
內容:
  - if / else      → 條件判斷（錯誤處理、模式切換）
  - for / while    → 迴圈（LED toggle N 次）
  - switch / case  → 多種狀態（不同 LED pattern）

關鍵字:範例
內容:
  - if (fd < 0) { ... }         → 錯誤就提早離開
  - for (i = 0; i < count; ++i) → 重複做同一件事
  - switch (state) { ... }      → 根據狀態選 pattern

------------------------------------------------------------
5) 檔案與模組層級
------------------------------------------------------------
關鍵字:.c / .h 檔案
內容:
  - .c: 實作（function body）
  - .h: 介面（function 宣告、struct 定義、enum）

關鍵字:API vs 實作細節
內容:
  - API: 給別的檔案用的函數（通常不加 static）
  - 實作細節: 只給自己這個 .c 用的 helper（static）

------------------------------------------------------------
6) 建置流程（Compile / Link）
------------------------------------------------------------
關鍵字:編譯
內容:
  - gcc -c xxx.c     → 產生 .o（object file）

關鍵字:連結
內容:
  - gcc xxx.o -o app → 把多個 .o 和函式庫黏在一起

關鍵字:結果
內容:
  - 所有「資料型態 + 變數 + 函數 + 控制流程」
    經過編譯/連結，變成一個可以執行的 binary

============================================================
關鍵字:程式組成摘要
內容:
  - 程式不是只有「code 行數」，而是：
      1) 資料型態（資料長相）
      2) 變數（實際存資料）
      3) 函數（封裝動作）
      4) 控制流程（決定路徑）
      5) 檔案/模組（API 與實作分離）
      6) 編譯/連結（組合成最終執行檔）
  - 寫 gpio_led_toggle 時：
      先想「要做哪些動作（函數）」，
      再回頭填「需要哪些資料型態與變數」，
      用 if/for 把流程串成一條完成的路。
